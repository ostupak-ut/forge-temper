# FORGE — Посібник для тестувальника 🇺🇦

**FORGE** — це візуальний конструктор потоків, який об’єднує ШІ-агентів (Claude Code, Codex, OpenRouter) на одному полотні. Ви перетягуєте вузли, з’єднуєте їх стрілками — і натискаєте «Запустити». Кожен вузол — це або агент (він реально працює з файлами на вашому комп’ютері), або дані.

Спеціалізація FORGE — автоматизація циклу **Forge → Temper**: один агент створює чернетку наукового прототипу (LaTeX + PDF), інший перевіряє його математику (алгебраїчно й чисельно) і повертає вердикт, а цикл повторюється, доки результати не «позеленіють». Але полотно універсальне — на ньому можна зібрати будь-який потік з власних агентів.

> ⏱️ **Найшвидший старт:** встановіть FORGE (Крок 1) → активуйте (Крок 2) → додайте **OpenRouter**-ключ (Крок 3, варіант В — лише ключ, без жодних CLI) → зберіть перший потік (Крок 4).

---

## Крок 1. Встановлення FORGE

1. Запустіть файл **`FORGE-Setup-0.1.0.exe`**, який вам надіслали.
2. ⚠️ Windows покаже синє вікно **«Windows захистив ваш ПК»** (застосунок ще не має цифрового підпису видавця — це нормально для тестової збірки). Натисніть **«Докладніше» → «Виконати в будь-якому разі»**.
3. У вікні встановлення оберіть теку (або лишіть за замовчуванням) → **«Встановити»**.
4. FORGE запуститься автоматично й додасть ярлик у **меню «Пуск»** та на робочий стіл. Надалі відкривайте його звідти — жодного терміналу не потрібно.

---

## Крок 2. Активація ліцензії

Ця збірка ліцензується окремо для кожного тестувальника й прив’язується до одного комп’ютера.

1. Після першого запуску ви побачите **екран активації** з вашим **ID пристрою**.
2. Натисніть **«Копіювати»** і надішліть цей ID тому, хто видав вам FORGE.
3. У відповідь вам надішлють **ключ ліцензії**. Вставте його у поле **«Ключ ліцензії»** → **«Активувати»**.
4. Застосунок завантажиться. Наступного разу просто відкривайте FORGE з меню «Пуск» — активація вже збережена.

> Ключ працює **лише на цьому комп’ютері** й має термін дії. Якщо ви зміните комп’ютер — попросіть новий ключ.

---

## Крок 3. Під’єднати ШІ-агентів

FORGE **не має власного ШІ** — він керує моделями через провайдерів, які працюють на вашому комп’ютері або через ваш API-ключ. Потрібен **щонайменше один** провайдер. Оберіть зручний (можна кілька):

| Провайдер | Що потрібно | Складність | Що дає |
|---|---|---|---|
| **OpenRouter** | лише API-ключ | ⭐ найпростіше | 400+ моделей: чат, генерація зображень/відео |
| **Claude Code** | встановити CLI + підписка Claude | ⭐⭐ | повноцінні агенти (навички **forge/temper**) — рекомендовано |
| **Codex** | встановити CLI + підписка ChatGPT | ⭐⭐ | агенти від OpenAI |

### Варіант В (найпростіший) — OpenRouter, лише ключ

Нічого встановлювати не треба — усе робиться всередині FORGE.

1. Зареєструйтеся на **https://openrouter.ai** і поповніть баланс (є й безкоштовні моделі).
2. Створіть ключ: **https://openrouter.ai/keys** → скопіюйте його (починається з `sk-or-...`).
3. У FORGE відкрийте **«Налаштування»** (внизу вікна) → блок **«API-ключі провайдерів»** → вставте ключ у поле **OpenRouter** → **«Зберегти ключі»**.
4. Тепер у будь-якому вузлі-агенті в полі **«Провайдер»** буде доступний OpenRouter.

> ⚠️ Моделі OpenRouter працюють у режимі **чату** — вони чудові для тексту/прози та генерації медіа, але **не запускають навички forge/temper** (для повного циклу перевірки потрібен Claude Code або Codex).

### Варіант А (рекомендований) — Claude Code

Дає найкращий результат: вузли **Forge** і **Temper** запускають однойменні навички як справжні агенти з доступом до файлів і терміналу.

1. Встановіть **Node.js** (LTS) з **https://nodejs.org** (просто «Далі → Далі»).
2. Відкрийте **PowerShell** (меню «Пуск» → напишіть `PowerShell`) і виконайте:
   ```powershell
   npm install -g @anthropic-ai/claude-code
   ```
3. Увійдіть у свій акаунт (потрібна підписка **Claude Pro/Max** або ключ Anthropic API):
   ```powershell
   claude
   ```
   Виконайте вказівки на екрані (відкриється браузер для входу). Коли побачите запрошення Claude — усе працює, можна закрити.
4. Перезапустіть FORGE. Він **автоматично** знайде `claude` — угорі має з’явитися зелений значок **«сервер працює · claude …»**.

### Варіант Б — Codex (ChatGPT)

1. Встановіть **Node.js** (як вище).
2. У PowerShell:
   ```powershell
   npm install -g @openai/codex
   ```
3. Увійдіть (потрібна підписка **ChatGPT Plus/Pro** або ключ OpenAI API):
   ```powershell
   codex login
   ```
4. Перезапустіть FORGE — він автоматично знайде `codex`. У вузлах з’явиться провайдер **Codex**.

> Якщо FORGE не бачить `claude`/`codex` після встановлення — відкрийте **«Налаштування» → «Шляхи до CLI»** і вкажіть шлях вручну (або просто перезапустіть застосунок, щоб оновився PATH).

## Крок 4. Ваш перший потік

### Інтерфейс
- **Ліворуч — палітра**: перелік вузлів. Клацніть, щоб додати на полотно, або перетягніть.
- **По центру — полотно**: тут ви з’єднуєте вузли.
- **Праворуч — властивості**: налаштування виділеного вузла (провайдер, модель, промпт).
- **Угорі — панель інструментів**: «Новий», «Шаблон», «Створити через ШІ», «Зберегти», **«Запустити граф»**.
- **Внизу** — «Файли» (Бібліотека + результати) і «Налаштування».
- **Мова**: перемикач **UA/EN** угорі праворуч, або **«Налаштування» → «Мова»**.

### Найшвидший спосіб — кнопка «Шаблон»
Натисніть **«Шаблон»** на панелі інструментів — FORGE збере готовий потік:

> **Картка → Цикл[Forge ↔ Temper] → Текст → Складання**

Далі:
1. Впишіть свою ідею у вузол-стікер (**«Промпт»**) — своє твердження, модель або результат простими словами.
2. У кожному агентному вузлі оберіть **«Провайдер»** (той, що ви налаштували в Кроці 3).
3. Натисніть **«Запустити граф»** — і спостерігайте, як стрілки оживають, а вузли по черзі виконуються.

### Основні поняття
- **Вузол (node)** — агент або дані. **Ребро (стрілка)** — переносить вихід одного вузла на вхід наступного. Порти з’єднуються лише за сумісних кольорів (типів).
- **Цикл** — з’єднайте вихід вузла назад у порт **feedback** ранішого вузла (напр. Temper → Forge). Рожева дуга «loop ≤N» означає, що цикл повторюватиметься, доки результат не пройде перевірку або не досягне межі ітерацій.
- **Пробний запуск** — прокручує потік і анімує стрілки **без** виклику агентів (безкоштовно, для перевірки з’єднань). **Запустити граф** — реальне виконання (витрачає токени/кредити).
- **▶ на вузлі** — запустити лише цей вузол (і те, від чого він залежить).

### Файли та результати
- **Бібліотека** — ваше сховище вхідних файлів. Завантажте файли/теки (або перетягніть їх прямо на полотно), потім оберіть їх у вузлі **«Файл»** і під’єднайте до агента.
- **Склад (Warehouse)** — під’єднайте його до виходу будь-якого агента: він збирає всі результати (PDF/MD/TEX/зображення/відео) в індексовану купу, що **накопичується** з кожним запуском.
- **Створити через ШІ** — опишіть потрібний потік звичайною мовою, і ШІ побудує граф за вас.

### Довідка в застосунку
Натисніть **«?»** угорі праворуч — там повна легенда: кольори портів, входи вузлів, як працюють цикли й запуск.

## Розв’язання проблем

| Симптом | Що зробити |
|---|---|
| Угорі «**сервер недоступний**» | Зачекайте кілька секунд після запуску; якщо не зникає — перезапустіть FORGE. |
| У вузлі немає потрібного провайдера | Ви не налаштували його в Кроці 3, або не перезапустили FORGE після встановлення CLI. |
| «сервер працює», але без «claude»/«codex» | CLI не в PATH: **«Налаштування» → «Шляхи до CLI»**, вкажіть шлях, або перезапустіть застосунок. |
| SmartScreen блокує інсталятор | **«Докладніше» → «Виконати в будь-якому разі»** (застосунок без підпису, але безпечний). |
| Модель відповідає з помилкою 500 / «overloaded» | У режимі **«Паралельно»** зменшіть кількість одночасних агентів (кнопка `×N`) до 1–2. |
